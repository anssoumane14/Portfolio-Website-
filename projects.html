    Telecom Customer Churn Analysis

    Situation : The telecom company faced a high churn rate, especially in cities like San Diego and Los Angeles. Reducing churn was key to preserving revenue and customer loyalty.

    Task : Analyze a real dataset to identify churn drivers, profile high-value customers, and generate actionable retention recommendations.

    Actions :

    ✔️ Cleaned and prepared dataset (7,043 rows) using Python (null handling, feature engineering, tenure groups).
    ✔️ Ran PostgreSQL queries to investigate churn patterns by contracts, offers, and demographics.
    ✔️ Created Power BI dashboards for churn visualization and segmentation.
    ✔️ Defined churn risk score on 18 variables and segmented high-risk customers.

    Results : Identified 45% churn due to competition and 17% due to dissatisfaction. 69% of high-value customers were exposed to risk. Retention strategies proposed.

    Challenge : Managing multi-source data and aligning Python, SQL, and Power BI pipelines.

    ReX : Gained end-to-end BI pipeline skills and communicated insights effectively to business teams.



    Smart Routing in Paris 11 (Dijkstra Algorithm)

    Situation : Needed to compute shortest pedestrian paths in the 11th arrondissement of Paris.

    Task : Design a program that calculates optimal routes using Dijkstra’s algorithm.

    Actions :

    Implemented the algorithm in Python and C.
    Visualized graphs and created an interactive path API.

    Results : Functional routing tool ready to be integrated into a web app for urban navigation.

    Challenge : Managing graph schema complexity and debugging shortest path performance.

    ReX : This project improved my algorithmic thinking and made me confident in API integration.
